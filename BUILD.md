# Building Stofloos locally

All builds are **local** — no Expo/EAS cloud. Mobile uses `expo prebuild` to generate native
projects, then Gradle (Android) / Xcode (iOS). Desktop uses electron-builder. The relay is a Docker
image.

Every client only needs one runtime config value: **`T3CODE_RELAY_URL`** (your relay's public
`https://` origin). Put it in the repo-root `.env` (or `.env.local`) — the build reads it via
`scripts/lib/public-config.ts` and injects it into each client.

```
# .env  (repo root)
T3CODE_RELAY_URL=https://relay.example.com
# optional client tracing:
# T3CODE_RELAY_CLIENT_OTLP_TRACES_URL=...
# T3CODE_MOBILE_OTLP_TRACES_URL=...
```

Install workspace deps once: `pnpm install`.

## Desktop (Electron) — Windows x64, macOS arm64

Uses the existing pipeline in `scripts/build-desktop-artifact.ts` (electron-builder). App id
`nl.stofloos`, product name **Stofloos**, artifacts named `Stofloos-<version>-<arch>.<ext>`.

**Prerequisites:** Node ≥ 22, plus electron-builder's platform tools. First run downloads Electron
(~200 MB) and the packaging toolchain, so ensure disk space.

**macOS arm64** (run on an Apple-Silicon Mac):

```sh
pnpm dist:desktop:dmg:arm64      # → dist/Stofloos-<version>-arm64.dmg
```

- Unsigned by default. For a signed/notarized build set `T3CODE_APPLE_TEAM_ID` (and, if using a
  provisioning profile, `T3CODE_MACOS_PROVISIONING_PROFILE`) and pass `--signed`.

**Windows x64** (run on Windows — cross-building from macOS/Linux is not supported here; there is
no `wine` fallback):

```sh
pnpm dist:desktop:win:x64        # → dist/Stofloos-<version>-x64.exe (NSIS installer)
```

- Optional Authenticode signing via Azure Trusted Signing env vars (see the build script).

**Linux x64** (AppImage): `pnpm dist:desktop:linux`.

The build bundles the CLI server (`apps/server`) and web renderer (`apps/web`) automatically.

## Mobile (Expo, local builds — no EAS)

Bundle/app ids: `nl.stofloos` (production), `nl.stofloos.dev`, `nl.stofloos.preview`. The native
`android/` and `ios/` folders are generated by `expo prebuild` (git-ignored).

Scripts live in `apps/mobile` — run with `pnpm --filter @t3tools/mobile <script>`.

### Android

**Prerequisites:** JDK 17 (Gradle for modern Android; the repo's Java 11 is too old for release
builds), Android SDK + platform-tools (`ANDROID_HOME` set), an Android emulator or device for
`run:android`.

```sh
# Debug build + install on a running emulator/device:
pnpm --filter @t3tools/mobile android:prod

# Release artifacts (no install):
pnpm --filter @t3tools/mobile build:android       # APK → apps/mobile/android/app/build/outputs/apk/release/
pnpm --filter @t3tools/mobile build:android:aab   # AAB → apps/mobile/android/app/build/outputs/bundle/release/
```

Release signing: generate a keystore and set `apps/mobile/android/gradle.properties`
(`MYAPP_UPLOAD_STORE_FILE`, `…_KEY_ALIAS`, `…_STORE_PASSWORD`, `…_KEY_PASSWORD`) — the release build
is unsigned/debug-signed until you configure this.

### iOS

**Prerequisites:** macOS + Xcode + CocoaPods, an Apple Developer account for device/store builds.

```sh
# Generate the native project, then build & run (Release) on a simulator/device:
pnpm --filter @t3tools/mobile build:ios

# Or just generate the project and open it in Xcode to archive/export an .ipa:
pnpm --filter @t3tools/mobile prebuild:ios
open apps/mobile/ios/Stofloos.xcworkspace
# Xcode → Product → Archive → Distribute App
```

Signing: set your Team in Xcode (Signing & Capabilities) or via `xcodebuild` `DEVELOPMENT_TEAM`.

## Relay (Docker → Docker Swarm)

The relay ships as a Bun + SQLite container. See `infra/relay/DEPLOY.md` for the full guide.

```sh
docker build -t stofloos-relay:latest -f infra/relay/Dockerfile .
docker stack deploy -c infra/relay/stack.yml stofloos     # on your Swarm
```

CI builds & pushes the image to GHCR on `main` (`.github/workflows/deploy-relay.yml`); point
`stack.yml`'s `image:` at the published tag to deploy.

## CI

- `.github/workflows/build.yml` — local builds on GitHub runners (desktop matrix + Android/iOS),
  no EAS, no Clerk.
- `.github/workflows/deploy-relay.yml` — builds & pushes the relay Docker image to GHCR.
- `.github/workflows/release.yml` is the upstream release pipeline (nightly scheduling, signing,
  npm publish, GitHub Releases). It still references the old Clerk/Cloudflare setup — adapt it to
  your release process (runners, signing secrets, registries) before using it.
